# Ejercicio: Fork, Branch y Pull Request

- **Objetivo:** Practicar el ciclo **Fork → Clone → Branch → Commit → Push → Pull Request (PR)**.
- **Interfaz:** Web (Fork, PR) y Desktop (Clone, Branch, Commit, Push).

1.  **Fork (Web):** Cada estudiante debe ir al repositorio principal y hacer un **Fork** a su propia cuenta de GitHub.
2.  **Clone (Desktop):** Cada estudiante debe clonar _su propio fork_ a su computadora local.
3.  **Crear Rama (Desktop):** Antes de hacer cambios, cada estudiante debe crear una nueva rama (branch) con un nombre descriptivo (ej: `mi-parrafo` o `aporte-juan-perez`).
4.  **Editar (Local):** El estudiante abre `HISTORIA.md` y añade _una sola frase_ (o un párrafo corto) que continúe la historia.
5.  **Commit y Push (Desktop):** El estudiante guarda el cambio, hace **commit** (con un mensaje claro, ej: "Añade mi frase a la historia") y hace **push** de su rama _a su fork_ (ej: `git push origin mi-parrafo`).
6.  **Crear Pull Request (Web):** El estudiante va a su fork en GitHub. Verá un aviso para crear un **Pull Request (PR)**. Debe crear el PR apuntando desde su rama (`mi-parrafo`) hacia la rama `main` del repositorio _original_ (el del profesor).
7.  **Revisión (Web):** ¡Aquí empieza la colaboración\! Antes de que el profesor acepte el PR, cada estudiante debe **revisar el PR de otro compañero**. Debe ir a la pestaña "Pull Requests" del repo original, elegir uno, y dejar un comentario (ej: "¡Buena continuación\!" o "Encontré un error ortográfico").
8.  **Merge (Web):** El profesor revisa los PRs comentados y los va "mergeando" (fusionando) uno por uno. La historia crecerá con los aportes de todos.
